\section{Introduzione}
% ============================================================
\begin{frame}{Introduzione}
  In ogni pipeline reale i dati non sono perfetti: misure imprecise, errori di
  annotazione, valori mancanti, anomalie.
  \vfill
  \begin{itemize}
    \item Non chiediamo quale modello sia più accurato sui dati puliti, ma quanto
          ciascuno resiste quando i dati si degradano.
    \item Corrompiamo il training set in modo progressivo con sei tipi di rumore,
          a tre livelli: 10\%, 25\%, 40\%.
    \item Tre modelli a confronto: rete neurale semplice (NN Base), rete
          regolarizzata (NN Pro), Random Forest.
    \item Metriche sempre sul test set pulito: accuratezza, F1 macro, AUC.
    \item Un esperimento non supervisionato (K-Means) chiude lo studio.
  \end{itemize}
  \vfill
  Seme fissato (\texttt{seed = 42}) su tutte le componenti stocastiche.
\end{frame}

\section{Dataset ed esplorazione}
% ============================================================
\begin{frame}{Il dataset MAGIC Gamma Telescope}
  \begin{columns}[c]
    \begin{column}{0.56\textwidth}
      \begin{itemize}
        \item Simulazioni Monte Carlo del telescopio Cherenkov MAGIC.
        \item Classificazione binaria: gamma (segnale) vs adroni (fondo).
        \item $19\,020$ osservazioni, $10$ feature continue.
        \item Classi sbilanciate: $64{,}8\%$ gamma, $35{,}2\%$ adroni.
        \item Riferimento: un classificatore banale arriva già al $\sim 65\%$.
      \end{itemize}
    \end{column}
    \begin{column}{0.44\textwidth}
      \centering
      \includegraphics[width=\linewidth]{eda_pie}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Suddivisione e preprocessing}
  \begin{itemize}
    \item Split stratificato $60/20/20$: $11\,412$ train, $3\,804$ val, $3\,804$ test.
    \item Analisi esplorativa e statistiche di preprocessing solo sul training set
          (niente \emph{data leakage}).
    \item Rimosse $41$ righe duplicate dal training.
    \item Standardizzazione (media $0$, deviazione $1$): un livello di rumore
          significa la stessa cosa per ogni feature.
  \end{itemize}
\end{frame}

\section{Modelli e baseline}
% ============================================================
\begin{frame}{I tre modelli}
  \begin{itemize}
    \item \textbf{NN Base}: Dense(32)$\to$Dense(16)$\to$sigmoide, nessuna
          regolarizzazione. Il modello ingenuo.
    \item \textbf{NN Pro}: stessa architettura $+$ L2, dropout $0{,}2$,
          class weight bilanciato, early stopping.
    \item \textbf{Random Forest}: $200$ alberi, aggregazione per voto.
  \end{itemize}
  \vfill
  Difese crescenti contro il rumore: ci aspettiamo NN Base la più esposta.
\end{frame}

\section{Metodologia del rumore}
% ============================================================
\begin{frame}{I sei meccanismi di rumore}
  \begin{itemize}
    \item \textbf{Feature noise}: rumore gaussiano sulle feature.
    \item \textbf{Label noise random}: etichette invertite a caso.
    \item \textbf{Label noise borderline}: etichette invertite sui campioni ambigui.
    \item \textbf{Missing values}: celle mancanti (NaN) da imputare.
    \item \textbf{Outlier}: valori estremi da trattare.
    \item \textbf{Missing feature}: sensori guasti, feature azzerate.
  \end{itemize}
  \vfill
  {\small Il rumore colpisce solo il training; validation e test restano puliti.
  Per ogni rumore riaddestriamo i tre modelli ai quattro livelli.}
\end{frame}

\section{Conclusioni}
% ============================================================
\begin{frame}{Conclusioni}
  \begin{itemize}
    \item I rumori hanno gravità diversa: outlier trascurabili, feature noise e
          valori mancanti gestibili, missing feature e label noise i più gravi.
    \item Il caso peggiore è il label noise borderline: falsifica il confine
          decisionale.
    \item Nessun modello domina: la NN Pro vince sul rumore delle etichette, il
          Random Forest su valori mancanti e outlier.
    \item Conta più il preprocessing del modello: imputare con la mediana,
          winsorizzare, fermare l'addestramento sulla validation recuperano molto.
    \item Nessun modello recupera un'informazione cancellata o falsificata nel
          punto più sensibile.
  \end{itemize}
\end{frame}
